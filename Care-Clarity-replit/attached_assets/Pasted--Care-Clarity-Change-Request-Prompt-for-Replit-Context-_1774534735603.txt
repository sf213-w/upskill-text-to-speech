# Care Clarity — Change Request Prompt for Replit

## Context
Care Clarity is an existing bilingual (Spanish ↔ English) medical translation PWA.

## Change 1 — Calendar & Appointment Reminder (fix `/book`)

### What to change
The "Add Spanish reminder to phone calendar" checkbox on `/book` is currently cosmetic.
Make it functional.

### What to build
- When the checkbox is checked and user selects a booking method, trigger a calendar
  event using the Web Calendar API
- Pre-fill the event with:
  - Title: "Cita con Dra. Ana López / Appointment with Dra. Ana López"
  - Location: "Centro Médico Salud"
  - Reminder: 1 hour before in Spanish —
    "Recordatorio: Cita con Dra. Ana López en 1 hora"
- If Web Calendar API is unavailable (iOS), fall back to a downloadable `.ics` file
- Save the reminder to localStorage under `careclarity_reminders`:
  ```json
  {
    "provider": "Dra. Ana López",
    "specialty": "Medicina General",
    "datetime": "[ISO timestamp]",
    "reminderSet": true
  }
  ```
- Add a "Mis Recordatorios / My Reminders" list to `/settings` that reads from
  `careclarity_reminders` and displays each saved reminder as a read-only row:
  - "Cita con Dra. Ana López · [date]" + "Delete" button
- Add i18n keys:
  - `booking.reminder_set` / `booking.reminder_failed`
  - `settings.reminders_title` / `settings.no_reminders`

## Change 2 — Consultation Transcript Clinical Metadata Filename (fix `/transcript/:id`)

### What to change
The auto-generated transcript filename currently uses only date + time.
Update it to include clinical metadata.

### What to build
- Update the auto-generated filename format to:
  `"[Date] · [Provider Name] · [Specialty] · #[Session ID]"`
  Example: `"2026-03-26 · Dra. Ana López · Medicina General · #4829"`
- Display a metadata header at the top of every transcript card:

  ```
  📅 Date:      March 26, 2026
  👩‍⚕️ Provider:  Dra. Ana López
  🏥 Specialty: Medicina General
  ⏱️ Duration:  12 min
  💬 Messages:  24
  ```

- Store this metadata in the session object in localStorage:
  ```json
  {
    "id": "4829",
    "provider": "Dra. Ana López",
    "specialty": "Medicina General",
    "date": "[ISO timestamp]",
    "duration": "[seconds]",
    "messageCount": 24,
    "messages": []
  }
  ```
- Update `storage.ts` — `saveSession()` to accept and store the new metadata fields
- Add i18n keys:
  - `transcript.provider` / `transcript.specialty`
  - `transcript.duration` / `transcript.messages`

## Change 4 — Full Legal Onboarding + Saved to Settings

### Part A — Update Onboarding Screen 3

Replace the single privacy notice line with a full scrollable legal acknowledgment screen:

- Screen title: "Legal & Privacy / Legal y Privacidad"
- Scrollable area containing 4 statements (each collapsible accordion):

  1. **Privacy Notice / Aviso de Privacidad**
     "🔒 This app does not save audio recordings. Transcriptions are
     stored only on your device."

  2. **HIPAA Rights Notice / Aviso de Derechos HIPAA**
     "You have the right to access, correct, and delete your health
     information at any time. This app does not share your information
     with third parties."

  3. **Terms of Use / Términos de Uso**
     "Care Clarity is a translation aid — not a medical device. Always
     confirm important medical information directly with your provider."

  4. **Data Deletion Rights / Derecho a Eliminar Datos**
     "You may delete all your saved data at any time from the Settings
     screen."

- Single acknowledge button at bottom:
  "Entiendo y acepto / I understand and agree"
  - Button is DISABLED until user has scrolled to the bottom of all statements
  - On tap → save to localStorage key `careclarity_legal`:
    ```json
    {
      "privacyAccepted": true,
      "hipaaAccepted": true,
      "termsAccepted": true,
      "dataRightsAccepted": true,
      "acceptedAt": "[ISO timestamp]",
      "acceptedLanguage": "en"
    }
    ```
  - Navigate to Home

### Part B — Add Legal & Consent Section to `/settings`

Add a new "Legal & Consent / Legal y Consentimiento" section to the Settings screen,
below the existing Security & Privacy section.

- Section title: "Legal & Consent / Legal y Consentimiento"
- 4 read-only rows — pull data from `careclarity_legal` in localStorage:

  | Icon | Statement | Badge | Action |
  |---|---|---|---|
  | 📄 | Privacy Notice / Aviso de Privacidad | Accepted ✓ [date] | [View] |
  | 🏥 | HIPAA Rights Notice / Aviso HIPAA | Accepted ✓ [date] | [View] |
  | 📋 | Terms of Use / Términos de Uso | Accepted ✓ [date] | [View] |
  | 🗑️ | Data Deletion Rights / Derecho a Eliminar | Accepted ✓ [date] | [View] |

- "Accepted ✓" badge is green; date shown in user's current language format
- "View" expands an inline accordion with the full text of that statement
- Statements are READ-ONLY — no toggles or checkboxes
- Accepted date pulled from `careclarity_legal.acceptedAt`
- Statement text displayed in `careclarity_legal.acceptedLanguage` by default
  with a small toggle to switch language inline

- Below the 4 rows add a "Delete All My Data" button:
  - Label: "Eliminar todos mis datos / Delete All My Data" (destructive red)
  - On tap → show confirmation modal:
    "¿Estás seguro? Esto eliminará todas tus sesiones, recordatorios y
    configuraciones guardadas. / Are you sure? This will delete all your
    sessions, reminders, and saved settings."
  - Confirm → clear ALL localStorage keys:
    `careclarity_sessions`, `careclarity_legal`, `careclarity_reminders`,
    `hasCompletedOnboarding`
  - Navigate back to onboarding Screen 1
